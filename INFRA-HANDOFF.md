# OpenClaw Infrastructure Repo — Handoff Document

This document contains everything needed to create and deploy the `openclaw-infra` repo in a new session. It is self-contained — no context from the current session is required.

---

## Context

There are two repos in this project:

| Repo | Purpose | Status |
|---|---|---|
| `Walsen/openclaw` | Builds and publishes the Docker image | ✅ Done — image at `ffactory/openclaw:latest` on Docker Hub |
| `Walsen/openclaw-infra` | Deploys the AWS infrastructure that runs the image | 🔲 To be created |

The infra repo deploys OpenClaw on **Amazon Bedrock AgentCore Runtime** using AWS CDK. It is based on the AWS sample [sample-host-openclaw-on-amazon-bedrock-agentcore](https://github.com/aws-samples/sample-host-openclaw-on-amazon-bedrock-agentcore), adapted to pull the pre-built image from Docker Hub instead of building it locally.

---

## Architecture

```
Telegram / Slack
      │
      ▼
API Gateway (HTTP API)
      │
      ▼
Router Lambda
  - validates webhook signature
  - resolves user identity (DynamoDB)
  - calls InvokeAgentRuntime
      │
      ▼
AgentCore Runtime  ◄── ffactory/openclaw:latest (ARM64)
  - per-user Firecracker microVM
  - OpenClaw Gateway (port 18789)
  - server.py HTTP contract (port 8080)
  - S3 workspace sync
      │
      ▼
Amazon Bedrock (Claude via ConverseStream)
```

**Key properties:**
- Per-user session isolation — each user gets their own microVM
- Serverless — microVMs spin up on message, idle-terminate after 30 min
- Persistent memory — workspace synced to S3 on shutdown, restored on cold start
- ARM64 (Firecracker) — image must be `linux/arm64`

## CDK Stacks

The deployment uses 8 CDK stacks deployed in 3 phases:

### Phase 1 — Foundation (no dependencies)

| Stack | Key Resources |
|---|---|
| `OpenClawVpc` | VPC, 2 AZs, private/public subnets, NAT gateway, 7 VPC endpoints, flow logs |
| `OpenClawSecurity` | KMS CMK, 7 Secrets Manager secrets, Cognito User Pool, optional CloudTrail |
| `OpenClawGuardrails` | Bedrock Guardrail (content filters, topic denial, PII redaction, word filters) |
| `OpenClawObservability` | CloudWatch dashboards, alarms, SNS, Bedrock model invocation logging |

### Phase 2 — AgentCore Runtime (AgentCore Starter Toolkit, not CDK)

| Resource | Notes |
|---|---|
| ECR repository | Created by toolkit |
| Docker image | **Pull from `ffactory/openclaw:latest`** — do NOT build locally |
| AgentCore Runtime | `CfnRuntime` — ARM64, private VPC, session timeouts |
| AgentCore Endpoint | `CfnRuntimeEndpoint` |

### Phase 3 — Application (depends on Phase 2 runtime_id)

| Stack | Key Resources |
|---|---|
| `OpenClawAgentCore` | IAM role, security group, S3 bucket (workspace), Workload Identity |
| `OpenClawRouter` | Router Lambda, API Gateway HTTP API, DynamoDB identity table |
| `OpenClawCron` | EventBridge Scheduler group, Cron executor Lambda |
| `OpenClawTokenMonitoring` | DynamoDB token usage table, Lambda processor, analytics dashboard |

## What to Change from the Sample Repo

The sample repo builds the Docker image locally during deployment. We skip that — the image is already published to Docker Hub.

### 1. `cdk.json` — pull from Docker Hub instead of building

The sample uses `image_version` to tag a locally-built image. Replace with a direct Docker Hub reference:

```json
{
  "context": {
    "account": "YOUR_AWS_ACCOUNT_ID",
    "region": "us-west-2",
    "docker_image": "ffactory/openclaw:latest",
    "default_model_id": "global.anthropic.claude-sonnet-4-6",
    "session_idle_timeout": 1800,
    "session_max_lifetime": 28800,
    "workspace_sync_interval_seconds": 300,
    "registration_open": false,
    "enable_guardrails": true,
    "guardrails_content_filter_level": "HIGH",
    "guardrails_pii_action": "ANONYMIZE",
    "enable_cloudtrail": false,
    "enable_browser": false,
    "router_lambda_timeout_seconds": 600,
    "router_lambda_memory_mb": 256,
    "cron_lambda_timeout_seconds": 900,
    "cron_lambda_memory_mb": 256,
    "daily_token_budget": 1000000,
    "daily_cost_budget_usd": 10,
    "token_ttl_days": 90,
    "user_files_ttl_days": 365,
    "cron_lead_time_minutes": 5,
    "cloudwatch_log_retention_days": 30
  }
}
```

### 2. `scripts/deploy.sh` — skip local Docker build

The sample's Phase 2 runs `agentcore deploy --local-build` which builds the image with Docker. Replace with `--image` to pull from Docker Hub:

```bash
# Phase 2: deploy runtime using pre-built image from Docker Hub
agentcore deploy \
  --image ffactory/openclaw:latest \
  --config .bedrock_agentcore.yaml
```

### 3. `.bedrock_agentcore.yaml` — auto-generated, but set image source

This file is auto-generated by `deploy.sh` from CDK outputs. The key field to ensure is set correctly:

```yaml
agent_runtime:
  image: ffactory/openclaw:latest   # ← Docker Hub, not ECR local build
  platform: linux/arm64
```

### 4. Environment variables injected into the Runtime

These are set by the CDK `AgentCoreStack` and passed to the container at runtime. Ensure these match what `server.py` expects:

```python
environment = {
    "S3_BUCKET":        s3_bucket.bucket_name,
    "STACK_NAME":       stack_name,
    "AWS_REGION":       region,
    "BEDROCK_MODEL_ID": cdk.json["default_model_id"],
    "GUARDRAIL_ID":     guardrail.ref,          # empty string if guardrails disabled
    "GUARDRAIL_VERSION": "DRAFT",
    "DYNAMODB_TABLE":   router_stack.table_name,
    "DYNAMODB_REGION":  region,
    # LOCAL_DEV must NOT be set in production
}
```

## Prerequisites

Install these before starting:

```bash
# AWS CLI v2
brew install awscli   # or see https://docs.aws.amazon.com/cli/latest/userguide/install-cliv2.html

# Node.js 24 + AWS CDK CLI
brew install node
npm install -g aws-cdk

# Python 3.11+
brew install python

# AgentCore Starter Toolkit
pip install bedrock-agentcore-toolkit

# Verify agentcore is on PATH
which agentcore   # if not found: export PATH="$HOME/.local/bin:$PATH"
```

## Step-by-Step Deployment

### 1. Create the repo

```bash
mkdir openclaw-infra && cd openclaw-infra
git init
git remote add origin git@github.com:Walsen/openclaw-infra.git
```

### 2. Clone the sample as the starting point

```bash
git clone https://github.com/aws-samples/sample-host-openclaw-on-amazon-bedrock-agentcore.git .
rm -rf .git
git init
```

### 3. Configure AWS credentials

```bash
aws configure   # or set AWS_PROFILE
aws sts get-caller-identity   # verify
```

### 4. Set account and region

```bash
export CDK_DEFAULT_ACCOUNT=$(aws sts get-caller-identity --query Account --output text)
export CDK_DEFAULT_REGION=us-west-2   # or us-east-1
```

Edit `cdk.json` — set `account` and `region` fields (see the modified `cdk.json` above).

### 5. Bootstrap CDK (first time only)

```bash
cdk bootstrap aws://$CDK_DEFAULT_ACCOUNT/$CDK_DEFAULT_REGION
```

### 6. Install Python dependencies

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 7. Validate CDK (runs cdk-nag security checks)

```bash
cdk synth
```

### 8. Deploy Phase 1 (foundation)

```bash
cdk deploy OpenClawVpc OpenClawSecurity OpenClawGuardrails OpenClawObservability \
  --require-approval never
```

### 9. Store Telegram bot token

Get a bot token from [@BotFather](https://t.me/BotFather) on Telegram, then:

```bash
aws secretsmanager update-secret \
  --secret-id openclaw/channels/telegram \
  --secret-string 'YOUR_TELEGRAM_BOT_TOKEN' \
  --region $CDK_DEFAULT_REGION
```

### 10. Deploy Phase 2 (AgentCore Runtime — pull from Docker Hub)

The sample's `deploy.sh` auto-generates `.bedrock_agentcore.yaml` from CDK outputs. Run it with the image override:

```bash
# Option A: use the sample's deploy script (recommended)
./scripts/deploy.sh --runtime-only

# Option B: manual agentcore deploy
agentcore deploy \
  --image ffactory/openclaw:latest \
  --config .bedrock_agentcore.yaml
```

After this completes, note the `runtime_id` and `runtime_endpoint_id` from the output. Add them to `cdk.json`:

```json
"runtime_id": "openclaw_agent-XXXXXXXXXX",
"runtime_endpoint_id": "DEFAULT"
```

### 11. Deploy Phase 3 (application stacks)

```bash
cdk deploy OpenClawAgentCore OpenClawRouter OpenClawCron OpenClawTokenMonitoring \
  --require-approval never
```

### 12. Set up Telegram webhook and allowlist

```bash
./scripts/setup-telegram.sh
```

This script:
1. Registers the Telegram webhook with API Gateway
2. Prompts for your Telegram user ID (get it from [@userinfobot](https://t.me/userinfobot))
3. Adds you to the DynamoDB allowlist

### 13. Test

Send a message to your Telegram bot. First message triggers a cold start (~10-15s for the lightweight agent, ~1-2min for full OpenClaw). Subsequent messages in the same session are fast.

## Key Configuration Decisions

### Channels

- **Telegram only** — simplest, run `setup-telegram.sh`
- **Telegram + Slack** — run both setup scripts; users can link accounts with `link` command
- Cross-channel account linking works out of the box once both are configured

### Guardrails

Enabled by default in `cdk.json` (`enable_guardrails: true`). Adds ~$0.75/1k text units.

To disable: set `"enable_guardrails": false` in `cdk.json` before deploying `OpenClawGuardrails`.

The container's `GUARDRAIL_ID` env var will be empty, and `server.py` skips guardrail checks when it's empty.

### Registration

`"registration_open": false` means only allowlisted users can register. Add users with:

```bash
./scripts/manage-allowlist.sh add telegram:YOUR_TELEGRAM_USER_ID
```

Set `"registration_open": true` to allow anyone.

### Region

AgentCore Runtime is available in: `us-east-1`, `us-west-2`, `eu-west-1`, `ap-northeast-1`.

If deployment fails with "unsupported availability zones", add to `cdk.json`:

```json
"availability_zones": ["us-east-1b", "us-east-1c"]
```

### Model

Default: `global.anthropic.claude-sonnet-4-6` — uses cross-region inference, routes to any available region automatically.

Cheaper alternative: `global.anthropic.claude-haiku-4-5-20251001-v1:0`

## Image Update Workflow

When `ffactory/openclaw` publishes a new image (on PR merge to `Walsen/openclaw` main):

```bash
# In openclaw-infra repo:
# 1. Update image tag in cdk.json if using a specific tag (not needed for :latest)
# 2. Redeploy AgentCore stack to pick up new image
cdk deploy OpenClawAgentCore --require-approval never

# New sessions will use the new image automatically
# Existing sessions continue until they idle-terminate
```

## Useful Operations

```bash
# Check runtime status
RUNTIME_ID=$(aws cloudformation describe-stacks \
  --stack-name OpenClawAgentCore \
  --query "Stacks[0].Outputs[?OutputKey=='RuntimeId'].OutputValue" \
  --output text --region $CDK_DEFAULT_REGION)
aws bedrock-agentcore get-runtime --agent-runtime-id $RUNTIME_ID --region $CDK_DEFAULT_REGION

# Check DynamoDB identity table
aws dynamodb scan --table-name openclaw-identity --region $CDK_DEFAULT_REGION

# Add user to allowlist
./scripts/manage-allowlist.sh add telegram:123456789

# View Router Lambda logs
aws logs tail /aws/lambda/openclaw-router --follow --region $CDK_DEFAULT_REGION

# Teardown everything
cdk destroy --all
# Note: KMS keys and Cognito User Pool have RETAIN policy — delete manually if needed
```
